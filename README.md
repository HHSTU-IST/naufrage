# 《海难迷梦》GDD

## 1. 项目概述

- **类型**：2D 单场景剧情推理 + 生存资源管理
- **极简描述** ：在15天食物耗尽前，辨认真实与梦境，从三位渔民互相矛盾的信息中推导唯一正确的求生路线。
- **核心卖点**
  - 单场景、低成本开发
  - 梦境真假推理
  - 信息比资源更重要
  - 多周目、多结局

## 2. 核心玩法循环（Core Loop）

``` text
 开始新的一天
      │
      ▼
   观察环境
(收集梦境线索)
      │
      ▼
  与渔民交流
      │
      ▼
   是否施救？
      │
      ▼
   选择行动
(停留/继续前进/返回)
      │
      ▼
   消耗食物
      │
      ▼
  进入下一天
```

玩家每天只完成一次循环。

## 3. 世界观与故事

- 玩家海难漂流到陌生海岸，与三名渔民一同幸存。
- 食物只能维持15天。
- 头部受伤导致每天醒来可能进入梦境。
- 梦境会改变环境与对话，但仍可能包含真实线索。
- 玩家必须通过推理，而不是运气，找到真正的村庄。

## 4. 七大核心系统

### 4.1 时间系统

- Day1\~Day15
- 每天结束自动推进

### 4.2 食物系统

每天：

- 生存 -1
- 施救 -1
- 行进 -1

Food=0

立即失败。

### 4.3 梦境系统

每天随机：

- Reality
- Dream

系统永不提示。

玩家通过观察判断。

示例：

  现实 梦境

  ---------- --------------

  太阳正常 太阳方向异常
  海浪正常 海浪静止
  脚印存在 脚印消失

### 4.4 NPC系统

三位渔民：

- A（西）
- B（北）
- C（东）

状态：

``` text
昏迷
 │
 ▼
苏醒
 │
 ▼
恢复
```

施救提升状态。

状态越高，信息越可靠。

### 4.5 信息系统

所有信息具有：

- 来源
- 日期
- 梦/现实
- 可信度（后台）

玩家自行建立推理。

### 4.6 路线系统

``` text
西：5天

北：8天

东：14天
```

每天：继续 或 返回

返回按已走距离原路返回。

### 4.7 结局系统

- 成功获救
- 饿死
- 时间耗尽
- 真结局

## 6. Demo设计

当前阶段只做最小可用版本（MVP），目标是“能从开局完整玩到结局”。

包含：

- 三位渔民
- 梦境判定
- 三条路线
- 食物消耗
- 存档/读档
- 至少两个可触发结局

预计10\~20分钟。

## 8. 玩家体验目标

- 前10分钟：学会观察。

- 20分钟：学会怀疑。

- 30分钟：建立自己的推理模型。

- 结局：玩家相信自己是因为推理正确，而不是随机运气获救。
